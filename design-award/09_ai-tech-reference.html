Flux Fill Pro v1
業界頂尖的文字引導影像 inpainting 模型，能將指定區域以提示詞描述的紋理替換。
🇩🇪 Black Forest Labs GmbH（德國） FLUX.1 [pro] Commercial
NT$1.5
/ 張 per generation
做什麼 What it does 將使用者上傳的設計圖 + 遮罩 + 文字 prompt 結合，AI 在遮罩區域內生成符合 prompt 描述的紋理。
何時觸發 When it fires 會員在 AI 設計工具按下「✨ 套用石材」按鈕時。輸出時間約 15–40 秒。
為何選擇 Why chosen 目前商業可用最強的 inpainting 模型，紋理寫實度與邊緣融合品質遠優於 Stable Diffusion Inpaint。
替代方案 Alternatives SDXL Inpaint（品質較弱）、DALL-E 3 Edit（無 mask 支援）、自建 ControlNet（部署複雜）。
設計圖 + 遮罩 + Prompt → Flux Fill Pro → 商業級寫實合成圖
👁
LLaVA v1.6 Mistral 7B
視覺語言大型模型，能「看懂」圖片並用自然語言描述，支援中英文輸出。
🇺🇸 University of Wisconsin–Madison + Microsoft Research Apache 2.0
NT$0.5
/ 張 per analysis
做什麼 What it does 看石材照片，輸出 10–15 個英文關鍵字描述（顏色、紋路、表面）作為 Flux Fill Pro 的 prompt。
何時觸發 When it fires 管理員上傳新石材照片時自動執行；或在編輯頁按「✨ 重新分析照片」手動觸發。
為何選擇 Why chosen 比規則式 prompt 模板精準 5–10 倍。LLaVA 已開源、Replicate 上易用、比 GPT-4V 便宜 80%。
替代方案 Alternatives GPT-4V（貴 5×）、Claude Vision（貴 4×）、BLIP-2（語意精準度較弱）。
石材照片 → LLaVA → "deep green serpentine with white veining, polished surface"

Pillow 11.x + pillow-heif 0.16+
Python 影像處理事實標準函式庫，加上 HEIC / HEIF 支援擴充。
🇺🇸 Alex Clark + Pillow 社群 MIT-CMU + Apache 2.0
免費
Open Source
做什麼 What it does 讀取上傳影像、轉換色彩模式、縮放、儲存；HEIC 自動轉成 JPG 確保瀏覽器相容。
何時觸發 When it fires 任何影像進入系統時：石材上傳、設計圖上傳、AI 合成回傳結果儲存。
為何選擇 Why chosen Python 生態系標準、pillow-heif 提供 iPhone 預設格式相容、零成本、無中資疑慮。
替代方案 Alternatives OpenCV（過於龐大）、Wand/ImageMagick（部署複雜）、ImageIO（HEIC 支援不完整）。
🔢
ImageHash 4.3+
感知雜湊（perceptual hash）演算法實作，產生影像的 64-bit 視覺指紋。
🇦🇹 Johannes Buchner（奧地利學者，個人開源專案） BSD-2-Clause
免費
Open Source
做什麼 What it does 計算每張石材照片的 pHash 雜湊值，比對時用 Hamming distance 算出視覺相似度。
何時觸發 When it fires 上架時建立索引；使用者用「以圖搜圖」時即時計算所有庫存的相似度排序。
為何選擇 Why chosen 無需訓練、零延遲、單張石材只需 64 bits 儲存。比 CNN-based embedding 簡單 100×。
替代方案 Alternatives CLIP embeddings（要 GPU、儲存大）、SIFT/SURF（已過時）、Resnet 特徵（部署複雜）。
參考圖 → pHash → 64-bit 指紋 → 相似度 0–100%
🌈
HSV 色彩空間分類 自建演算法
用 Python 標準庫 colorsys 將 RGB 轉 HSV，依色相/飽和度/亮度規則分類 8 種顏色。
🇺🇸 Python Software Foundation（colorsys 為標準庫） PSF License
免費
Stdlib
做什麼 What it does 分析石材主色，自動分類為白/灰/黑/米/棕/綠/紅/其他 8 種顏色標籤。
何時觸發 When it fires 上架時自動執行（如管理員未手動指定顏色）；批次重新索引時可一鍵重算。
為何選擇 Why chosen 石材色彩語意（如「綠色系」）用 HSV 比 RGB 直觀精準。針對蛇紋石擴展綠色 hue 到 195°。
替代方案 Alternatives K-means 主色提取（複雜度高）、CNN 分類（要訓練資料）、外部 API（不必要）。

三層 Prompt 工程系統
智能 prompt 生成機制，平衡「自動化」與「人工控制」的混合策略。
🇹🇼 ReStone 自建（台灣） Proprietary
免費
In-house
做什麼 What it does 為 Flux Fill Pro 產生最佳 prompt。優先順序：管理員手動編輯 → LLaVA 視覺 AI 寫描述 → 規則模板生成。
何時觸發 When it fires 設計師按「套用石材」按鈕時，系統依優先順序自動選用最精準的 prompt。
為何選擇 Why chosen 純依賴 AI 不可控、純依賴人工不自動化。三層 fallback 確保系統健壯（即使 LLaVA 失敗仍可運作）。
設計師可介入 Override 設計工具中可即時編輯 prompt（不存檔），管理員可永久編輯每個石材的 prompt。
石材屬性 → [Tier 1] 人工手動 → [Tier 2] LLaVA AI → [Tier 3] 模板規則 → 最終 Prompt
💬
中英同義詞語義搜尋字典
針對石材產業詞彙的中英對應與口語別名，純 Python dict 實作。
🇹🇼 ReStone 自建（台灣） Proprietary
免費
In-house
做什麼 What it does 設計師打「marble」或「大理石」或「雲石」都能搜到同樣結果。展開後送入 SQL LIKE 查詢。
範例 Examples 大理石 → [marble, 雲石, 大理岩, marmo]
花崗岩 → [granite, 花崗石, 麻石]
為何選擇 Why chosen 石材分類有限（4 大類），用字典精準度遠勝向量搜尋。零延遲、零成本、易維護。
替代方案 Alternatives Sentence Transformers embedding（過度設計）、Elasticsearch 同義詞索引（部署成本高）。

Replicate Python SDK 0.32+
統一介面呼叫雲端 AI 模型，封裝 Flux Fill Pro 與 LLaVA 的 API 通訊。
🇺🇸 Replicate, Inc.（舊金山，Y Combinator 投資） Apache 2.0
免費
SDK 免費，按模型用量付費
做什麼 What it does 將 ReStone 後端的 Python 函式呼叫，轉換為 Replicate 雲端的 HTTP API 請求。
何時觸發 When it fires 每次需要呼叫 Flux Fill Pro 或 LLaVA 時自動處理上傳檔案、輪詢結果、下載回傳。
為何選擇 Why chosen 一個 token 涵蓋所有開源 AI 模型，避免直接管理 GPU / 模型部署的複雜度。
替代方案 Alternatives 自建 GPU 伺服器（人月成本高）、Hugging Face Inference（部分模型缺）、fal.ai（生態較新）。
